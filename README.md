# Yatube API

## Описание проекта

**Yatube API** — REST‑API для социальной сети, где пользователи могут:
* Управление постами: создание, чтение, обновление и удаление записей.
* Работа с комментариями: добавление, просмотр, редактирование и удаление комментариев к постам.
* Подписки: управление подписками на авторов (создание и просмотр списка).
* Сообщества: просмотр списка сообществ и детальной информации о них.
* Аутентификация: получение, обновление и проверка JWT‑токена для авторизации.

Проект демонстрирует построение полноценного API на Django REST Framework с аутентификацией через JWT. Может использоваться как основа для фронтенд‑приложений или как учебный пример.

## Стек технологий

* **Python 3.x** — основной язык программирования.
* **Django 5.1.1** — веб‑фреймворк.
* **Django REST Framework 3.15.2** — построение API.
* **djangorestframework_simplejwt 5.4.0** — JWT‑аутентификация.
* **Djoser 2.3.1** — удобные эндпоинты для авторизации/регистрации.
* **Pillow 11.0.0** — работа с изображениями.
* **SQLite** (по умолчанию) / **PostgreSQL** — база данных.
* **pytest 8.3.3** — тестирование.
* **flake8 7.1.1** — проверка стиля кода.

### Как развернуть проект:

##### Клонировать репозиторий 
```
git clone <https://github.com/GoodWinee/yatube-API.git>
```
##### Cоздать и активировать виртуальное окружение:
```
python -m venv venv
source venv/Scripts/activate
```
##### Установить зависимости из файла requirements.txt:
```
pip install -r requirements.txt
python -m pip install --upgrade pip
```
##### Выполнить миграции:
```
python manage.py migrate
```
##### Запустить проект:
```
python manage.py runserver
```

### Примеры запросов:

##### Получение списка постов:
```
GET /api/v1/posts/
```
##### Создание поста (авторизация обязательна):
```
POST /api/v1/posts/
{
  "text": "Это мой новый пост",
  "group": 1,  // ID группы (опционально)
  "image": "url_или_файл"  // при необходимости
}
```
##### Получение комментариев к посту:
```
GET /api/v1/posts/<post_id>/comments/
```
##### Подписка на пользователя:
```
POST /api/v1/follow/
{
  "following": "username_подписчика"
}
```
